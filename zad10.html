1. Wymień oraz opisz dwie części selektora.

Selektor w CSS składa się z dwóch głównych części:

a) Element docelowy: Określa typ elementu HTML, na którym chcemy zastosować styl. Może to być tag HTML, klasa, identyfikator lub nawet pseudoelement. Na przykład, jeśli chcemy stylizować wszystkie nagłówki pierwszego poziomu <h1>, elementem docelowym będzie h1.

b) Deklaracje stylów: Są to właściwości CSS, które chcemy zastosować do wybranych elementów docelowych. Mogą to być np. kolor tła, rozmiar czcionki, marginesy itp. Na przykład, jeśli chcemy ustawić czerwony kolor tekstu dla nagłówków pierwszego poziomu, deklaracją stylu może być color: red.

2. Znaczenie nawiasów klamrowych {...}

Nawiasy klamrowe {} w CSS są używane do grupowania deklaracji stylów i tworzenia bloków kodu. W kontekście selektora, nawiasy klamrowe otaczają deklaracje stylów i definiują zakres ich działania dla wybranego elementu docelowego. Wszystkie deklaracje stylów znajdujące się wewnątrz nawiasów klamrowych będą miały zastosowanie do wybranych elementów.

Przykład:
```css
h1 {
color: red;
font-size: 24px;
}
```

W powyższym przykładzie nawiasy klamrowe {} grupują deklaracje color: red; i font-size: 24px; jako część selektora h1. Oznacza to, że wszystkie nagłówki pierwszego poziomu (<h1>) na stronie będą miały czerwony kolor tekstu oraz rozmiar czcionki 24 piksele.

3. Opisz cechy selektorów

Selektory w CSS mają różne cechy, które decydują o sposobie, w jaki elementy są wybierane i stylizowane. Oto niektóre z podstawowych cech selektorów:

- Specyficzność: Określa, jak dokładnie i precyzyjnie selektor wybiera elementy docelowe. Selektory o wyższej specyficzności mają większy priorytet w zastosowaniu stylów. Na przykład, selektor zawierający klasę (.) ma większą specyficzność niż selektor tagu (div).

- Dziedziczenie: Niektóre style mogą być dziedziczone przez elementy potomne. Oznacza to, że jeśli nadamy styl pewnemu elementowi, jego elementy potomne dziedziczą ten styl, chyba że zostanie nadpisany innym stylem.

- Kaskadowość: CSS używa zasady kaskadowości, która określa, jakie style zostaną zastosowane do elementów, gdy istnieje kilka reguł dotyczących tego samego elementu. Style zdefiniowane wewnętrzniej (np. w arkuszu stylów) mają większy priorytet niż style zdefiniowane zewnętrznie (np. w pliku CSS zewnętrznym).

- Możliwość zagnieżdżania: CSS umożliwia zagnieżdżanie selektorów, co oznacza, że można określać style dla elementów znajdujących się wewnątrz innych elementów. Na przykład, możemy zastosować styl tylko do linków znajdujących się wewnątrz pewnego kontenera.

4. Opisz wartości selektorów

Selektory w CSS mają różne wartości, które określają, jakie elementy zostaną wybrane i stylizowane. Oto trzy podstawowe wartości selektorów:

1. Tag HTML: Selektor tagu jest najprostszym rodzajem selektora i odnosi się do wszystkich elementów o określonym tagu HTML. Na przykład, selektor div odnosi się do wszystkich elementów <div>.

2. Klasa: Selektor klasy jest poprzedzony kropką (np. .klasa) i odnosi się do elementów, które mają określoną klasę. Można nadawać jednej lub wielu elementom tę samą klasę, aby zastosować do nich wspólne style.

3. Identyfikator: Selektor identyfikatora jest poprzedzony znakiem hash (np. #identyfikator) i odnosi się do elementu, który ma określony identyfikator. Każdy identyfikator powinien być unikalny na stronie, ponieważ może być stosowany tylko do jednego elementu.

5. Kiedy stosujemy znak średnika (;), kiedy nie jest konieczny

W CSS znak średnika (;) jest używany do oddzielenia kolejnych deklaracji stylów w bloku deklaracji. Stosujemy go po każdej deklaracji stylu, aby wskazać, że jedna deklaracja się kończy i kolejna się zaczyna.

Przykład zastosowania znaku średnika (;):
```css
h1 {
color: red;
font-size: 24px;
}
```

W powyższym przykładzie każda deklaracja (color: red; i font-size: 24px;) jest zakończona znakiem średnika (;), aby oddzielić je od siebie.

Niektóre przypadki, kiedy znak średnika (;) nie jest konieczny:
- Jeśli w bloku deklaracji jest tylko jedna deklaracja stylu, to znak średnika (;) nie jest wymagany.
- Jeśli ostatnia deklaracja w bloku deklaracji ma znak } (zamykający nawias klamrowy), to znak średnika (;) po niej nie jest konieczny.
